Samesite Cookies are a way to determine when a site's cookies are present in a request from another site. Chrome came up with *Lax*, which is the fallback security restriction set if a website does not have its own restriction system. Most browsers are expected to adapt this.

**Top Level Domain (TLD):** This is the .com, .gov, etc ( topmost domain level )
**Top Level Domain + 1 (TLD+1):** This is the TLD, along with one additional level of the domain name ( google.com )
 **Effective Top Level Domain (eTLD):** Accounts for multipart suffixes like .co.uk
 **Scheme**: http://, https://
 **Port**
 **Subdomain:** www, other subdomains of the main domain ( like *mail*.google.com )
 **Domain name:** Everything except scheme, subdomain and port in the URL

SameSite system works by limiting what kind of cookies can come in from external **sites**.
Owners can make certain cookies valid only if the origin is from the same site.


### Strict
This allows NO inter-site cookie transfer at all. Undesirable when cross-site functionality is desired, but is the most secure

### Lax
Cookies are sent only if two conditions are met.
1. The request must be a GET request
2. The request must have originated from a top-level user action, such as clicking a link. This means the exploits through scripts, image sources, etc. will not work.

### None
No restrictions, worth investigating
