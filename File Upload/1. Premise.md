# Possible Weaknesses:
### 1. Forms with files:
a. Normal forms are sent as URL encoded data. For others that include files, "multipart/form-data" is preferred as content-type (in header)
Sample: 
```
`POST /images HTTP/1.1 
Host: normal-website.com 
Content-Length: 12345 
Content-Type: multipart/form-data; 
boundary=---------------------------012345678901234567890123456 
---------------------------012345678901234567890123456 
Content-Disposition: form-data; 
name="image"; 
filename="example.jpg" 
Content-Type: image/jpeg 
[...binary content of example.jpg...] 
---------------------------012345678901234567890123456 
Content-Disposition: form-data; 
name="description" 
This is an interesting description of my image. 
---------------------------012345678901234567890123456 
Content-Disposition: form-data; 
name="username" wiener 
---------------------------012345678901234567890123456--`
```

If the server trusts just the content-type without validating the binary, easily exploitable through Burp Repeater/intruder - just upload the file, and change content type to a trusted one.
