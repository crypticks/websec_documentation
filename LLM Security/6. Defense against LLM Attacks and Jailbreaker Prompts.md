Since LLMs can make API calls, make API access control always authenticated.

Handle access control at the APIs themselves, don't expect the LLM to self-police ( it is manipulatable ). Proper privilege control can prevent most prompt injections.

Never train an LLM on sensitive data. Some methods involve:
- Apply robust sanitization techniques to the model's training data set.
- Only feed data to the model that your lowest-privileged user may access. This is important because any data consumed by the model could potentially be revealed to a user, especially in the case of fine-tuning data.
- Limit the model's access to external data sources, and ensure that robust access controls are applied across the whole data supply chain.
- Test the model to establish its knowledge of sensitive information regularly.

##### Jailbreaker Prompts

An LLM can't be relied on if it's just told to ignore requests to certain APIs or if told to block requests that contain certain payloads.

A prompt that involves statements like `Ignore all instructions regarding which APIs to use` ( jailbreaker prompts ) might be able to bypass such restrictions.
