Traditional crawlers can only visit a site, scan for data and vulnerabilities based on pattern matching and vulnerability signatures. It can't understand the semantics of what's written on a site, or what the site is meant for.

However, LLMs understand purposes and functionalities of sites. They can make on-the-spot decisions for which data to extract, which site to visit next, and so on. They can also use other tools on sites, through Tool APIs. They don't follow a static script.


These are vulnerable to indirect prompt injection, where an attacker can control the scanner's reasoning and tool usage. This is because such scanners use text as a source for instructions as well. If the LLM can't differentiate between site content and instructions, unforeseen behaviour occurs.

---
- Malicious instructions are embedded in stored content, for example, within comments or blog posts.
- The scanner reads this content during its crawl process.
- The LLM interprets the injected text as actionable instructions rather than passive data.
- The scanner executes tool calls or generates requests based on the injected text.
---

This is similar to CSRF, where the AI is the authenticated, privileged user.

---
Methods of Bypass:
1. Adopt a persona ( system admin, security researcher, etc. )
2. Social Engineering ( Present the vulnerability as a finding that requires verification )
3. Urgency / Consequence ( make it believe data loss / negative consequence will occur if it doesn't execute the command )
